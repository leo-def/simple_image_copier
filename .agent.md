# AI Agent Configuration for Simple Image Copier

> Configuration and capabilities for AI-assisted development.

## 🤖 Agent Capabilities

### Code Understanding
- **Java Concurrency**: Threads, ExecutorService, synchronization primitives
- **Image Processing**: BufferedImage, pixel manipulation, ImageIO
- **Thread Pools**: Work distribution, task scheduling
- **Performance**: Thread overhead, memory usage, optimization
- **Error Handling**: Exception handling in threaded contexts

### Development Tasks Agent Can Perform

✅ **Can Help With**:
- Implementing thread pool architecture
- Creating pixel processing tasks
- Implementing image loading/saving
- Optimizing thread work distribution
- Writing concurrent unit tests
- Performance profiling and optimization
- Memory usage analysis
- Error handling in threaded contexts
- Parallel image processing algorithms
- Output format conversions

❌ **Not Recommended**:
- Using too many threads without performance analysis
- Direct pixel manipulation without buffering
- Hardcoding thread count
- Unsafe concurrent access to BufferedImage
- Processing without proper exception handling

## 🛠 Recommended Skills

### Skill 1: Thread Pool Setup
**When**: Initializing concurrent image processing
**Process**:
1. Create ExecutorService with fixed thread pool
2. Calculate work chunks (rows or columns)
3. Submit PixelProcessor tasks
4. Wait for completion with shutdown/awaitTermination
5. Handle InterruptedException gracefully

### Skill 2: Pixel Processing Task
**When**: Implementing worker thread logic
**Process**:
1. Extend Runnable with pixel processing logic
2. Calculate assigned row/column range
3. Iterate through pixels in range
4. Copy pixel values from source to destination
5. Handle exceptions in run()

### Skill 3: Performance Optimization
**When**: Tuning thread count and chunk size
**Process**:
1. Profile with different thread counts
2. Measure total processing time
3. Monitor CPU utilization
4. Check memory usage
5. Find optimal configuration

---

## 🔑 Key Concepts

- **ExecutorService**: Manages thread lifecycle and tasks
- **Work Partitioning**: Divide image into chunks per thread
- **Synchronization**: Safe concurrent access to shared resources
- **Thread Pool Size**: Usually CPU cores for optimal performance
- **Pixel Access**: Efficient row-by-row processing
