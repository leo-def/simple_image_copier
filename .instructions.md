# Development Instructions for AI Agents - Simple Image Copier

> Guidelines for AI-assisted development of the multi-threaded image processing utility.

## 🎯 Project Overview

- **Purpose**: Copy images using parallel pixel processing
- **Type**: Command-line utility
- **Tech Stack**: Java 8+, java.util.concurrent, ImageIO
- **Build**: Maven or Gradle

---

## 🏗 Architecture Pattern

**Thread Pool + Worker Pattern**:
- Main thread loads image
- ExecutorService creates N worker threads
- Each thread processes a row/column partition
- Main thread waits for completion and writes output

### Core Components
```
ImageCopier      - Main orchestrator
PixelProcessor   - Runnable worker task
ImageLoader      - I/O utilities
ThreadPool       - ExecutorService wrapper
```

---

## 📂 File Naming & Conventions

**Classes**: PascalCase with descriptive names
- `ImageCopier.java`, `PixelProcessor.java`, `ImageLoader.java`

**Methods**: camelCase
- `copyWithThreads()`, `processPixels()`, `loadImage()`

**Variables**: camelCase
- `threadCount`, `sourceImage`, `startRow`, `endRow`

**Constants**: UPPER_SNAKE_CASE
- `MAX_THREADS`, `DEFAULT_THREAD_COUNT`, `BUFFER_SIZE`

---

## ✅ DO's

1. **DO** use ExecutorService for thread management
2. **DO** partition image intelligently (by rows preferred)
3. **DO** synchronize shared resources properly
4. **DO** test with different thread counts
5. **DO** validate input image format before processing

---

## ❌ DON'Ts

1. **DON'T** create more threads than CPU cores + 1
2. **DON'T** access BufferedImage pixels without synchronization
3. **DON'T** hardcode thread count (make configurable)
4. **DON'T** assume square images
5. **DON'T** process without error handling

---

## 📋 Common Workflows

**Multi-threaded image processing**:
1. Load image with ImageIO.read()
2. Create ExecutorService with thread pool
3. Calculate row/column chunks per thread
4. Create PixelProcessor tasks for each chunk
5. Submit tasks to executor
6. Wait for completion with awaitTermination()
7. Write output with ImageIO.write()

**Optimizing thread count**:
1. Get CPU core count: `Runtime.getRuntime().availableProcessors()`
2. Use thread count = cores (for I/O bound) or cores + 1 (for CPU bound)
3. Test performance with different counts
4. Monitor memory usage and thread overhead

**Handling large images**:
1. Load incrementally if possible
2. Use stripwise processing (process rows at a time)
3. Monitor heap memory usage
4. Consider swap file limitations
